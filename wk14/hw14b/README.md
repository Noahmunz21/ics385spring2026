This assignment builds a standalone Express.js authentication application using Passport.js, bcrypt, and MongoDB Atlas. The app has six routes: GET /register renders a registration form where new users enter their email and password; POST /register saves the new user to MongoDB with their password automatically hashed by bcrypt before storing (via a Mongoose pre-save hook); GET /login renders the login form; POST /login uses Passport.js LocalStrategy to verify the user's email and hashed password, redirecting to /profile on success or back to /login on failure; GET /profile is a protected route only accessible to authenticated users, displaying their email and role — unauthenticated users are immediately redirected to /login; and GET /logout clears the session and redirects back to /login. Sessions are managed with express-session, keeping the user logged in across page visits without re-entering credentials. AI Tools Used: Claude (Anthropic) assisted with Passport.js configuration, bcrypt pre-save hook, and connect-mongo session store debugging.

Bcrypt protects user passwords by running them through a one-way hashing algorithm with a cost factor (10 salt rounds in this project), making it computationally expensive to reverse or brute-force. Even if the database is compromised, attackers only see the hash — never the original password. Each hash also includes a unique random salt, meaning two users with the same password will have completely different hashes. This prevents rainbow table attacks where precomputed hash lists are used to crack passwords.
