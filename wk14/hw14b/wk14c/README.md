The most significant authentication design decision was using email as the login field instead of a traditional username, configured through Passport.js LocalStrategy's usernameField option. This was chosen because the property admin is likely to remember their email address more reliably than a separate username, and it aligns with modern web application conventions. Using bcrypt with 10 salt rounds for password hashing was equally critical — it ensures that even if the MongoDB database is compromised, passwords remain protected.
